# 6G Federated TinyML: Phased Array Beamforming Tracking

Данный проект реализует распределенную систему федеративного машинного обучения (Federated Learning) на микроконтроллерах ESP32 для задач предсказания пространственной траектории движения абонента в сетях 6G.

Система позволяет в реальном времени собирать данные с инерциальных датчиков (IMU), выполнять их обработку, прогнозировать угловое перемещение и локально дообучать нейронную сеть с использованием оптимизатора Adam. Локальные веса периодически отправляются на центральный сервер по протоколу MQTT для проведения процедуры глобального усреднения (FedAvg).

## Аппаратное обеспечение
- **Микроконтроллер**: ESP32 (Wi-Fi, I2C, Serial)
- **Датчик**: LSM6DS3 (Акселерометр + Гироскоп, 6 осей). Подключается по I2C (адрес `0x6A` или `0x6B`).

## Архитектура системы

Проект построен на базе **Split-Architecture (Разделенная архитектура)**:
1. **Замороженный Backbone (Извлекатель признаков)**: Модель TFLite (`model_cut.h`), загружаемая в оперативную память ESP32. Выполняет сверточные преобразования над времéнными окнами сырых данных.
2. **Обучаемый Слой (Trainable Head)**: Один полносвязный слой (`head_weights.h`), который инициализируется случайными весами и обучается напрямую на устройстве средствами языка C++.

### Основные программные особенности
* **Оптимизатор Adam**: Интегрирован алгоритм Adam с динамическим затуханием шага обучения (Learning Rate Decay) для быстрого схождения весов.
* **On-the-fly Нормализация**: Реализован алгоритм экспоненциально взвешенного скользящего среднего (EWMA) для стандартизации входных потоков гироскопа и акселерометра в реальном времени, что предотвращает насыщение градиентов без затрат оперативной памяти.
* **Потоковая передача MQTT**: Для обхода системных лимитов библиотеки `PubSubClient`, массив локальных весов отправляется на сервер с помощью стримингового API (beginPublish/write/endPublish).

## Компоненты проекта

### 1. Прошивка ESP32 (`inference.ino`)
Основной цикл устройства. Выполняет:
- Опрос датчика LSM6DS3 с частотой 100 Гц.
- Стандартизацию данных.
- Инференс Backbone-сети через `TensorFlow Lite Micro`.
- Обучение Head-выхода с использованием функции потерь MSE.
- Синхронизацию весов через MQTT (`broker.emqx.io`).
- Выдачу телеметрии в Serial-порт для локальной визуализации.

### 2. Python Визуализатор (`visualizer_6g.py`)
Интерфейс для мониторинга в реальном времени. Читает данные из USB/COM-порта устройства и строит графики для:
- Интегрированной траектории движения (Pitch/Yaw).
- Траектории, оцененной нейросетью.
- Оценки бимформинга антенной решетки: мгновенную пространственную метрику ошибки в градусах (с целевой зоной захвата `< 5°`).

### 3. Федеративный Сервер (`federated_server.py`)
Сервисный скрипт на Python, который подписывается на MQTT-топик агрегации (`6g_lab/+/local_weights`).
- Собирает массивы весов (Trainable Head) от разрозненных ESP32.
- Усредняет полученные веса (алгоритм Federated Averaging).
- Может транслировать усредненную глобальную модель обратно на устройства (Global Sync).

## Запуск проекта

1. **Сервер**: Запустите `python federated_server.py` на ПК или облачном сервере для прослушивания MQTT-канала.
2. **Прошивка**: 
   - Настройте `WIFI_SSID` и `WIFI_PASS` в файле `config.h`. 
   - Загрузите `inference.ino` в ESP32 через Arduino IDE.
3. **Визуализация**: Подключите ESP32 по USB к ПК, закройте Serial Monitor в Arduino IDE и запустите `python visualizer_6g.py` (убедившись, что в коде скрипта верно указан COM-порт).
